# Foundry Toolkit 中的模型playground
Foundry Toolkit playground提供了一个交互式环境，用于体验生成式AI模型。你可以测试各种提示，调整模型参数，比较不同模型的响应，并通过附加不同类型的输入文件来探索多模态能力。

![Foundry Toolkit playground界面，展示与生成式AI模型的聊天交互。](./images/playground/model-playground.png)

## 在playground中测试模型

要访问playground：

- 在Foundry Toolkit视图中，选择**开发者工具** > **构建** > **模型Playground**
- 从模型目录中的模型卡片上选择**在Playground中试用**
- 从**我的模型**中双击一个模型，将其在playground中打开

要在playground中测试模型，请按照以下步骤操作：

1. 在**模型首选项**中，从下拉列表中选择一个模型。
2. （可选）添加**系统提示**来引导模型响应。
3. （可选）配置所选模型可用的模型参数。
4. 在聊天输入框中输入聊天提示。

### Playground中的本地模型执行

Foundry Toolkit支持在playground中运行本地模型。你可以从**模型首选项**面板的模型下拉列表中选择本地模型。

![本地模型支持的截图](./images/playground/local-model-support.png)

## GitHub 按需付费模型支持

Foundry Toolkit现在支持GitHub按需付费模型，因此你可以在超过免费层限制后继续工作。

- 当你达到GitHub的模型使用限制时，Foundry Toolkit会显示一条警告，并提供指向GitHub付费使用文档的链接。

    ![GitHub按需付费模型支持的截图](./images/playground/github-pay-as-you-go.png)

- 在[GitHub设置](https://github.com/settings/billing)中开启模型的计费功能（需要登录你的GitHub帐户）。

- 启用计费后，无需更改工作流程即可继续在Playground或Agent Builder中使用该模型。

## 模型首选项

![模型首选项面板，显示生成式AI模型的可调节参数](./images/playground/parameters.png)

**模型首选项**面板允许你选择用于playground交互的模型，设置系统提示以引导模型的行为，并为所选模型配置模型参数。

### 选择模型
**模型首选项**面板中的下拉列表显示你已添加到Foundry Toolkit的模型。你可以选择**浏览模型**打开模型目录并添加更多模型。

### 系统提示
系统提示是一个预定义的指令，用于引导模型的行为和响应风格。你可以在输入文本区域中添加、编辑或删除系统提示，以影响模型生成响应的方式。

### 配置推理参数
可用的参数取决于模型类型和发布者。常见参数包括：
- Temperature（温度）：控制模型输出的随机性。较低的值使输出更具确定性。较高的值引入更多可变性。
- Top P：通过限制所选标记的累积概率来控制输出的多样性。较低的值产生更聚焦的响应。较高的值允许更多样化的输出。
- 最大响应长度（标记数）：设置模型响应中的最大标记数。这限制了生成文本的长度。
- 频率惩罚：降低模型在响应中重复相同标记的可能性。较高的值抑制重复。
- 存在惩罚：鼓励模型在响应中引入新主题或概念。较高的值促进生成文本的多样性。

### 显示资源使用情况

对于本地模型，还有一个额外的选项**显示资源使用情况**。启用后，每个响应都会显示资源使用情况。

![模型首选项面板中显示"显示资源使用情况"的截图](./images/playground/parameters-show-resource-usage.png)


## 聊天提示

![Foundry Toolkit playground中的聊天提示输入框，显示用于添加文件的附件图标。](./images/playground/chat-prompt.png)

聊天提示是你提供给模型的输入文本。你可以输入问题、陈述或任何你希望模型响应的文本。模型根据提供的提示以及已配置的系统提示和参数生成响应。

聊天提示输入框中可用的功能：
- **发送**：将聊天提示提交给模型进行处理。
- **清除**：清除聊天提示输入框。
- **文件附件**：为多模态模型附加文件到聊天提示。
- **图像附件**：为多模态模型附加图像到聊天提示。
- **网络搜索**：执行网络搜索以查找与聊天提示相关的信息。
- **模式切换**：在不同模式之间切换以使用AI功能。

### 切换模式
你可以在聊天提示输入框中切换不同的模式：
- **使用模型原生支持的AI功能**：此模式允许你使用所选模型的原生能力，例如网络搜索或文件附件。
- **使用所有AI功能**：此模式允许你使用Foundry Toolkit实现的AI功能，例如无论模型的原生能力如何都可以附加文件。

如果某项能力不被所选模型或Foundry Toolkit支持，则会在聊天提示输入框上方出现警告。该警告建议你切换到支持该功能的其他模型。

![截图，显示聊天提示输入框上方出现的警告消息，内容为"不支持该能力"。](./images/playground/capability-warning.png)

## 模型响应
![Foundry Toolkit playground响应区域，显示一个用于对文本进行base64编码的生成Python代码片段，并提供复制或重新生成响应的选项。](./images/playground/response-area.png)

模型响应显示在聊天提示输入框上方。在你输入聊天提示之前，playground会显示一个欢迎屏幕，其中包含三个入门提示供你开始使用playground。

模型响应区域中可用的功能：
- **复制响应**：将模型的响应文本复制到剪贴板。
- **重新生成响应**：基于相同的提示重新生成模型的响应，或使用另一个模型重新生成响应。
- **复制代码**：将响应中生成的代码复制到剪贴板或插入到新文件中。
- **渲染**：Playground支持渲染代码片段、图像、Markdown、LaTeX以及模型响应中的其他内容。
- **标记计数**：显示模型响应中使用的标记数。这有助于你了解模型的成本和性能。

### 资源使用情况

当本地模型启用了**显示资源使用情况**时，每个响应都会显示使用情况摘要。你还可以通过选择**使用详情**切换到[性能分析](/docs/intelligentapps/profiling)详情。

![显示一个响应的资源使用情况的截图](./images/playground/response-resource-usage.png)

## 工具栏

playground中的工具栏提供了对各种操作和功能的快速访问：
![Foundry Toolkit playground中的工具栏，显示用于清除聊天历史记录、保存聊天记录和切换模式的按钮。](./images/playground/tool-bar.png)

- **新建Playground**：创建一个新的playground会话。
- **历史记录**：查看、重命名、删除和切换聊天历史记录。
- **自动命名**：聊天会话的名称根据用户提示自动生成。
- **比较**：并排比较不同模型的响应。
- **查看代码**：查看LLM集成中为聊天提示生成的代码。

### 比较模型响应
![截图，显示在Foundry Toolkit playground中并排比较不同模型响应的效果。](./images/playground/compare.png)

**比较**功能允许你并排比较不同模型的响应。这对于评估不同模型对相同提示的性能非常有用。

要比较模型响应：
1. 选择工具栏中的**比较**按钮。
2. 从下拉列表中选择你要比较的模型。
3. 在其中一个输入框中输入相同的聊天提示。你的提示会被复制到其他输入框中。
4. 选择**发送**将提示提交给所有选定的模型。
5. 每个模型的响应会并排显示，便于比较。

对于你正在比较的每个模型，你可以：
- 将响应文本复制到剪贴板。
- 为所选模型配置模型参数。
- 继续与所选模型进行聊天。

## 你学到了什么

在本文中，你学会了如何：

- 使用Foundry Toolkit playground与生成式AI模型进行交互。
- 在playground中测试模型，配置系统提示和参数，并通过附加文件探索多模态能力。
- 使用**比较**功能评估不同模型对相同提示的性能，并并排查看响应。
- 使用聊天提示输入框发送提示、附加文件或图像、执行网络搜索以及在AI功能的不同模式之间切换。
- 查看模型响应，复制或重新生成响应，并渲染代码片段、图像、Markdown和LaTeX。
- 跟踪模型响应中的标记计数以了解成本和性能。
- 使用模型首选项面板选择模型、设置系统提示和调整推理参数。
- 使用工具栏创建新的playground会话，查看和管理聊天历史记录，自动命名会话，比较模型以及查看生成的代码。
- 管理聊天历史记录，包括查看、重命名、删除和切换会话。
