# JavaScript 性能分析

Visual Studio Code 支持收集和查看 JavaScript 程序的性能分析数据。要开始使用，您首先需要为 [Node.js 程序](/docs/nodejs/nodejs-debugging.md)或[浏览器应用](/docs/nodejs/browser-debugging.md)配置好调试器。

## 分析类型

根据您想要优化的内容，可能需要收集不同类型的分析数据。

* **CPU 分析** - CPU 分析可以告诉您程序在 JavaScript 中将时间花在了哪里。请记住，虽然您可以等待异步 Promise 或回调，但同一时刻只能有一个 JavaScript 表达式在执行。CPU 分析每秒大约采集 10,000 个样本，以告诉您当前正在执行哪个表达式（如果有的话）。
* **堆分析** - 堆分析可以告诉您程序中的内存是如何随时间分配的。分配内存可能开销很大，减少代码分配的内存量可以提升性能。
* **堆快照** - 堆快照是程序内存分配情况的即时视图。如果您发现程序使用了大量 RAM 并想找出来源，堆快照会很有用。请注意，对于更复杂的程序，收集堆快照可能需要数分钟，并且内置编辑器目前不支持查看堆快照。

## 收集分析数据

要收集分析数据，您需要先进入程序调试状态。进入调试后，您可以通过以下几种方式收集分析数据：

* 使用**调用堆栈**视图中的"录制"按钮。这可以获取 CPU 分析、堆分析和堆快照。
* 在程序中调用 `console.profile()`。这将收集 CPU 分析。

### 使用录制按钮

进入调试后，切换到"运行和调试"视图（`kb(workbench.view.debug)`），找到**调用堆栈**视图。将鼠标悬停在要调试的会话上，然后选择**获取性能分析**按钮。您也可以通过命令面板（`kb(workbench.action.showCommands)`）运行此命令。

![显示按钮的截图](images/profiling/take-profile.png)

然后 VS Code 会询问您想要哪种[性能分析类型](#types-of-profiles)；选择适合您的类型。

最后，VS Code 会询问您想要何时停止分析。您可以选择以下任一项：

* 持续分析直到手动停止。
* 分析一段设定好的时长。
* 分析直到命中某个断点。

如果您选择了第一个选项，可以通过点击调试工具栏中显示的大红色"录制"图标来停止分析。分析数据收集完成后，[分析查看器](#analyzing-a-profile)将自动打开。

### 使用 console.profile

您可以使用 `console.profile` 调用来手动在代码中添加分析标记以启动分析，并使用 `console.profileEnd` 来停止分析。CPU 分析数据将在这两个调用点之间收集。

```js
console.profile();
doSomeVeryExpensiveWork();
console.profileEnd();
```

生成的 `.cpuprofile` 文件将自动保存在您的工作区文件夹中。您可以选择该文件在内置的[分析查看器](#analyzing-a-profile)中打开它。

## 分析分析数据

### 表格视图

VS Code 带有一个集成的可视化查看器，支持查看 JavaScript 的 `.cpuprofile` 和 `.heapprofile` 文件。当您打开这些文件时，首先会看到一个类似下表的表格视图：

![显示分析表格的截图](images/profiling/profile-table.png)

这是一个**自底向上**的程序视图。每一行代表程序中的一个函数，默认按照在该特定函数中花费的时间排序。这也称为"自身时间"。函数的"总时间"是在该函数及其调用的所有函数中花费的时间之和。您可以展开每个表格行来查看该函数是从哪里被调用的。

例如，以下代码：

```js
function a() {
  doSomethingFor5Seconds();
  b();
}

function b() {
  doSomethingFor3Seconds();
}

a();
```

在这个例子中，`a` 的自身时间为 5 秒，总时间为 8 秒。`b` 的自身时间和总时间均为 3 秒。堆分析的工作方式相同，但使用自身大小和总大小来表示在每个函数或其被调用函数中分配的内存量。

### 火焰图视图

表格视图在某些情况下很好用，但通常您可能希望看到更直观的分析数据表示。您可以通过点击表格视图右上角的火焰 🔥 图标来实现这一点。如果您尚未安装提供火焰图编辑器的附加扩展，系统会提示您安装。

![堆分析火焰图](images/profiling/heap-profile.png)

乍看之下可能有些令人困惑，但别担心，我们来理解一下！

对于 CPU 分析，横轴是分析的时间线，让您可以查看程序在每个时刻的执行情况。对于堆分析，横轴是程序分配的总内存。

图表中的每个条形（或"火焰"）代表一个调用堆栈。最外层的顶级函数调用（或者说调用堆栈的"底部"）显示在编辑器顶部，它调用的函数显示在下方。条形的宽度由其总时间或总内存决定。

您可以点击火焰图中的某个条目查看更多信息，也可以使用鼠标滚轮进行缩放。放大后，您还可以在图表上任意拖拽来进行导航。

### 左倾视图

如果您正在处理 CPU 分析，您看到的火焰图很可能不像上面那个那样容易理解。您可能会看到大量看起来像这样的独立调用堆栈：

![杂乱的 CPU 分析](images/profiling/cpu-profile-noise.png)

为了便于分析，VS Code 提供了"左倾"视图，将所有相似的调用堆栈分组在一起。

![切换按钮](images/profiling/toggle-button.png)

这将从按时间顺序的分析视图切换为更类似堆分析的视图。横轴仍然是总分析时长，但每个条形代表该函数在所有从该堆栈调用的次数中的总时间。

![更清晰的 CPU 分析](images/profiling/cpu-profile-nice.png)

对于某些应用（如服务器），单个调用可能相对较快，但您想找出在多次调用中总体开销最大的函数，这种视图会更有用。
