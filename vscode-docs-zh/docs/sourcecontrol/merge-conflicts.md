# 在 VS Code 中解决合并冲突

当 Git 无法自动合并来自不同分支的更改时，就会发生合并冲突。VS Code 提供了多种工具来帮助你高效地识别和解决这些冲突。

了解如何使用内联编辑器操作、三向合并编辑器以及 AI 辅助解决 Visual Studio Code 中的合并冲突。

## 理解合并冲突

当 Git 遇到无法自动解决的竞争性更改时，就会发生合并冲突。常见场景包括：

* 两个分支修改了文件中的同一行
* 一个分支删除了另一个分支正在修改的文件
* 两个分支在同一位置添加了不同的内容

当你尝试合并、变基、拉取或拣选包含冲突更改的提交时，Git 会暂停操作并标记冲突，以便你手动解决。

> [!TIP]
> 详细了解[创建和管理分支](/docs/sourcecontrol/branches-worktrees.md)，以规划你的开发工作并减少合并冲突。

## 在 VS Code 中识别冲突

当合并冲突发生时，VS Code 会提供多种视觉指示器来帮助你识别和解决它们。有冲突的文件会显示在源代码管理视图的 **合并更改（Merge Changes）** 部分下。

### 编辑器冲突标记

当你打开有冲突的文件时，VS Code 会使用以下标记高亮显示冲突部分：

* `<<<<<<< HEAD`（或当前分支名称）：标记当前分支更改的开始
* `=======`：分隔两个冲突版本
* `>>>>>>> branch-name`：标记传入分支更改的结束

![显示当前和传入更改的内联冲突标记截图。](images/overview/merge-conflict.png)

### CodeLens 操作

在每个冲突上方，VS Code 会显示 CodeLens 操作，让你可以快速解决冲突：

* **接受当前更改**：仅保留你所在分支的版本
* **接受传入更改**：仅保留传入分支的版本
* **接受两者更改**：同时保留两个版本，依次排列
* **比较更改**：打开差异视图并排查看差异

## 使用内联操作解决冲突

对于简单的冲突，你可以直接使用编辑器中 CodeLens 操作来解决：

1. 从源代码管理视图中打开有冲突的文件

1. 查看标有冲突标记的冲突部分

1. 选择冲突上方的 CodeLens 操作之一：
   * **接受当前更改** 保留你的版本
   * **接受传入更改** 保留传入版本
   * **接受两者更改** 同时保留两个版本

1. 对文件中的每个冲突重复此操作

1. 所有冲突解决后保存文件

在文件中解决冲突后，冲突标记会消失，你可以暂存该文件，文件将移至源代码管理视图中的 **已暂存更改（Staged Changes）** 部分。然后你可以继续提交更改。

> [!TIP]
> 对于需要合并两者更改的较复杂冲突，你可以手动编辑文件。删除冲突标记（`<<<<<<<`、`=======`、`>>>>>>>`）并编辑内容以创建所需的结果。

## 使用三向合并编辑器

对于更复杂的冲突，或者当你想要并排查看所有更改时，可以使用三向合并编辑器。此编辑器提供了一个包含三个面板的全面视图：

* **传入（Incoming）**（左侧）：来自被合并分支的更改
* **当前（Current）**（右侧）：来自你当前分支的更改
* **结果（Result）**（底部）：将被保存的合并结果

### 打开合并编辑器

要打开三向合并编辑器：

1. 在源代码管理视图中选择有冲突的文件

1. 从上下文菜单中选择 **在合并编辑器中打开**，或选择编辑器顶部出现的 **在合并编辑器中解决** 按钮

    ![显示传入、当前和结果面板的三向合并编辑器截图。](images/overview/merge-editor-overview.png)

### 在合并编辑器中接受更改

合并编辑器会在每个冲突更改旁边显示复选框。使用这些复选框可以选择要接受哪些更改：

1. 查看 **传入** 和 **当前** 面板中的更改

1. 使用编辑器 CodeLens 操作选择要接受哪些更改：

   * 接受 **传入** 或 **当前** 更改
   * 接受 **两者的组合** 更改，以智能方式合并两者
   * **忽略** 更改，将其排除在结果之外

1. 底部的 **结果** 面板会自动更新以显示你的选择

1. 使用结果编辑器右侧的冲突计数指示器跟踪还有多少未解决的冲突。

1. 当所有冲突解决后，选择 **完成合并** 来暂存更改并关闭合并编辑器

### 在合并编辑器中进行手动编辑

有时你需要合并两者的部分更改或创建一个全新的解决方案。你可以直接编辑 **结果** 面板：

1. 在 **结果** 面板中任意位置点击以放置光标

1. 根据需要编辑内容，结合两个更改中的元素或编写新代码

> [!TIP]
> 使用 **传入** 或 **当前** 面板中的 **忽略** 按钮来隐藏你不想接受的更改。这有助于聚焦于相关冲突。

### 替代布局

选择合并编辑器右上角的三个点（···）可以打开包含其他选项的上下文菜单。你可以切换到垂直布局并显示基础视图，该视图显示在进行任何更改之前文件的状态。

**传入**、**当前** 和 **结果** 旁边的三个点提供了每个视图的选项，例如显示与基础的并排差异、接受所有更改或重置结果。

## 使用 AI 解决冲突（实验性功能）

> [!NOTE]
> AI 辅助冲突解决是一项实验性功能，需要 GitHub Copilot 订阅。

VS Code 可以[使用 AI](/docs/setup/copilot.md) 帮助自动解决合并冲突。此功能会分析两个冲突更改的版本并提出解决方案：

1. 在编辑器中打开有冲突的文件

1. 选择编辑器顶部的 **使用 AI 解决合并冲突** 按钮

    ![编辑器中 AI 合并冲突解决按钮的截图。](images/overview/ai-merge-conflict-resolution.png)

1. VS Code 会打开聊天视图并启动一个代理流程来分析冲突

1. 在编辑器中查看建议的更改

1. 接受解决方案或根据需要手动调整

AI 会考虑合并基础（两个分支的共同祖先）以及每个分支的更改，生成一个力求保留两个更改意图的解决方案。

## 完成合并操作

解决所有冲突后，你需要通过提交更改来完成合并操作：

1. 验证所有冲突文件都已解决并位于 **已暂存更改** 部分

1. 输入提交信息并选择 **提交** 来完成合并

> [!TIP]
> 如果你需要放弃合并操作，请从命令面板中选择 **Git: Abort Merge**。这会将你的仓库恢复到合并开始之前的状态。

## 将 VS Code 配置为默认合并工具

你可以通过在终端中运行以下命令，将 VS Code 设为默认的 Git 合并工具。当你在命令行中执行合并操作并产生冲突时，Git 会打开 VS Code 的合并编辑器来帮助你解决冲突。

```bash
git config --global merge.tool vscode
git config --global mergetool.vscode.cmd 'code --wait $MERGED'
```

要将 VS Code 设为 Git 的默认差异工具，请运行：

```bash
git config --global diff.tool vscode
git config --global difftool.vscode.cmd 'code --wait --diff $LOCAL $REMOTE'
```
