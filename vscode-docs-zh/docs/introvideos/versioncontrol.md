# VS Code 中的版本控制

在本教程中，我们将学习如何在 Visual Studio Code 中使用 Git 版本控制的基础知识。有关集成 Git 支持的更多信息，包括如何使用远程仓库，请继续阅读下方的[相关资源](#related-resources)部分。

## 视频大纲

* 安装 Git
  * [https://git-scm.com](https://git-scm.com)
* 打开文件夹
  * **文件** > **打开文件夹** (`kb(workbench.action.files.openFolder)`)
* 源代码管理
  * **查看** > **源代码管理 (SCM)** (`kb(workbench.view.scm)`)
* 初始化仓库
  * `main` 是默认分支
* 打开命令面板。
  * **查看** > **命令面板** (`kb(workbench.action.showCommands)`)
* 重命名分支
  * **Git: 重命名分支**
* 文件版本控制状态
  * **U** - 未跟踪的文件
  * **A** - 已添加的文件
  * **M** - 已修改的文件
* 提交文件
  * **提交** ☑️（复选标记）按钮
* 创建分支
  * **Git: 创建分支**
* 差异编辑器
  * **内联视图**按钮
* 暂存更改
  * **暂存更改** ➕ 按钮
* 切换分支
  * 状态栏分支项（左下方）
* 合并分支
  * **视图和更多操作** (**...**) > **分支** > **合并分支**
* 将分支发布到 GitHub
* 克隆仓库
  * **Git: 克隆** > **从 URL 克隆**
