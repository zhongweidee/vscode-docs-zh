# 重构 C++ 代码

Visual Studio Code 中的 C/C++ 扩展提供了多种重构功能，帮助您在保持运行时行为不变的前提下，改善代码的结构、可读性和可维护性。这些功能包括提取方法（Extract Method）和创建声明与定义（Create Declaration and Definitions）等。

## 创建声明或定义

让 C/C++ 扩展为您自动生成函数的声明或定义，从而简化创建过程。此功能适用于成员函数、命名空间类以及模板。

![跨两个文件创建定义和声明](images/refactoring/create-declaration-and-definition-different-files.gif)

要创建函数声明或定义，可以采用以下任一方式：

* 选中您的类函数定义，选择代码操作（灯泡图标），然后选择**创建声明**。同样，要创建定义，请选中函数声明，选择代码操作，然后选择**创建定义**。
* 右键点击函数的声明或定义，从上下文菜单中选择**创建定义/声明**。根据您的代码情况，系统会自动创建定义或声明。
* 选中函数，然后使用**命令面板**（`kb(workbench.action.showCommands)`）并输入命令**创建声明/定义**。根据您的代码情况，系统会自动创建定义或声明。

定义和声明的创建位置基于您之前在代码中建立的习惯模式。例如，如果您之前在同一文件中添加了定义和声明，那么扩展也会将新的内容添加到同一个文件中。函数的排列顺序会自动保持。

![在同一文件中创建定义和声明](images/refactoring/create-declaration-and-definition-same-file.gif)

如果您之前将声明或定义放在了与源文件不同的文件中，扩展会遵循您已有的惯例。例如，对于名称匹配的头文件和源文件，即使头文件尚未被包含在源文件中，两者也会被识别。在这种情况下，一旦添加了声明或定义，头文件会被自动引用到您的源文件中。

否则，扩展会为您创建一个与当前文件名称匹配的新头文件或源文件。这个新文件包含已生成的新声明或定义，然后会自动引用到您的当前文件中。

对于模板，如果函数模板在头文件中声明，则该函数模板的定义会在同一个头文件中创建。这同样适用于类模板的非模板成员函数。

### 复制声明或定义

如果您想要在代码中自由选择声明或定义添加的位置，可以使用代码操作**复制声明/定义**。这会将声明或定义复制到剪贴板，而不是直接添加到代码中。

要调用代码操作，请选中具有快速修复（Quick Fix）可用的函数，然后选择代码操作（灯泡图标），并选择**复制‘YourFunctionName’的定义**或**复制‘YourFunctionName’的声明**。

![复制声明或定义](images/refactoring/copy-declaration-definition.gif)

## 提取到方法

提取方法（Extract Method）重构功能允许您将一段代码提取到一个独立的方法中，以提高代码可读性、减少重复，并使代码更加模块化。

要提取方法，请选中您要提取的 C++ 代码，选择代码操作（灯泡图标），然后选择**提取到函数**。或者，右键点击代码并选择**重构 > 提取**，或使用键盘快捷键（Windows 上为 `Ctrl + Shift + R, Ctrl + E`）获取更多信息。

然后，您可以为创建的新函数命名。包含您所选代码的新函数会被放置在当前函数的上方。

![提取方法并创建声明](images/refactoring/extract-method.gif)

## 快速修复/代码操作

C/C++ 扩展会根据您的代码上下文，提供 C/C++ 专属的建议来帮助您修复和改进代码。您可以通过将鼠标悬停在符号上并选择**快速修复**链接，或者选择代码旁边出现的代码操作（灯泡图标）来查看这些建议。例如，如果一段代码可以被提取为方法，选择灯泡图标就会显示“提取到方法”。除上述功能外，C/C++ 扩展还在以下情况中提供快速修复/代码操作：

### 添加缺失的头文件

如果您的 C++ 代码中存在未知符号，并且 C/C++ 扩展在工作区中识别到了正确的头文件，则现在会有一个快速修复可用。选择该快速修复，必要的头文件包含语句将被添加至当前 C++ 文件的顶部。
