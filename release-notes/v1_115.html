![工具调用截图，显示"在浏览器中点击了元素"。](images/1_115/browser-tool-old.webp) _新版:_
![工具调用截图，显示"在测试页中右键点击了标题横幅"，并带有指向测试页的链接。](images/1_115/browser-tool-new.webp) #### 长时间运行的脚本支持 `运行 Playwright 代码` 工具改进对长时间运行脚本的支持。运行时间超过 5 秒的脚本（默认情况下）现在会返回一个延迟结果供智能体轮询。 #### 减少重复标签页 现在更强烈地阻止智能体重复打开浏览器标签页。当智能体尝试打开新标签页，且已有一个标签页打开到相同的主机时，除非智能体显式传递了标志，否则不会打开新标签页。 ### 集成浏览器中的双指缩放（macOS） [集成浏览器](https://code.visualstudio.com/docs/debugtest/integrated-browser) 现在在 macOS 上支持双指缩放手势。使用触控板双指手势可以将网页内容放大至 3 倍。 与标准浏览器缩放（`kb(workbench.action.browser.zoomIn)` / `kb(workbench.action.browser.zoomOut)`）不同，双指缩放是纯粹的可视化放大，不会重新排版页面布局。 ## 终端工具改进 此版本改进了智能体在后台运行终端命令的体验。 ### 向后台终端发送输入 此前，后台终端是只读的，仅支持 `get_terminal_output`。这尤其限制了当前台终端超时并移至后台时，智能体无法再与其交互的情况。 借助新的 `send_to_terminal` 工具，智能体可以继续与后台终端交互。例如，如果 SSH 会话在等待密码提示时超时，智能体仍然可以发送所需的输入来完成连接。 ### 后台终端通知（实验性） **设置**: `setting(chat.tools.terminal.backgroundNotifications)` 此前，当终端命令在后台运行时，智能体必须手动调用 `get_terminal_output` 来检查其状态。无法知道命令何时完成或需要输入。 通过新的实验性 `setting(chat.tools.terminal.backgroundNotifications)` 设置，当后台终端命令完成或需要用户输入时，智能体会自动收到通知。这也适用于超时并移至后台的前台终端。然后，智能体可以采取适当的操作，例如审查输出或通过 `send_to_terminal` 工具提供输入。 ## 已弃用的功能和设置 ### 此版本中新增的弃用项 无 ### 即将到来的弃用项 * **编辑模式** 自 VS Code 1.110 版本起已正式弃用。用户可以通过 VS Code 设置 `setting(chat.editMode.hidden)` 临时重新启用编辑模式。此设置将一直支持到 1.125 版本。从 1.125 版本开始，编辑模式将被完全移除，不再能通过设置启用。 ## 值得注意的修复 * [vscode#304257](https://github.com/microsoft/vscode/issues/304257) - 集成 pwsh
