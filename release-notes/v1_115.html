Visual Studio Code 1.115

关注我们的 LinkedIn、X、Bluesky

_发布日期: 2026年4月8日_

欢迎来到 Visual Studio Code 1.115 版本。此版本通过引入全新的 VS Code Agents 配套应用，让您的智能体原生开发体验更加出色！

  • VS Code Agents 应用：一款全新的配套应用，针对智能体原生开发进行了优化，与 VS Code Insiders 并行运行。
  • 集成浏览器：多项改进，使智能体使用集成浏览器更加无缝。
  • 终端工具：智能体与后台终端交互的新功能。

祝您编程愉快！

Visual Studio Code Agents（预览版）

Visual Studio Code Agents 是一款全新的预览版配套应用，随 VS Code Insiders 一起发布，专为智能体原生开发而构建。

  • 跨项目并行执行任务 - 在多个仓库之间并行启动智能体会话（每个会话都在自己的工作树中隔离运行），快速切换上下文（界面会根据您的选择自适应调整），并在人工审查和智能体审查之间进行迭代。
  • 监控和审查 - 跟踪会话进度，以内联方式查看差异，为智能体提供反馈，无需离开应用即可创建拉取请求。
  • 自定义配置无缝继承 - 自定义指令、提示文件、自定义智能体、MCP 服务器、钩子和插件都可以在 Agents 应用中使用，同时还包括您的其他 VS Code 自定义设置，例如主题。
  • 无需额外安装 - 该应用随 VS Code Insiders 一起提供。从操作系统的开始菜单或应用程序文件夹启动它，或者从命令面板运行聊天: 打开 Agents 应用程序。

Agents 应用是一个快速迭代的预览版。它目前仅在 VS Code Insiders 中可用，我们期待在 GitHub 问题中收到您的反馈。

集成浏览器

在此版本中，我们继续进一步增强集成浏览器体验及其对智能体的支持能力。

浏览器智能体工具改进

设置: setting(workbench.browser.enableChatTools)

更好的工具标签

当智能体调用浏览器工具时，工具调用现在具有更具描述性的标签和一个直接跳转到目标浏览器标签页的链接。

_旧版:_

_新版:_

长时间运行的脚本支持

运行 Playwright 代码 工具改进对长时间运行脚本的支持。运行时间超过 5 秒的脚本（默认情况下）现在会返回一个延迟结果供智能体轮询。

减少重复标签页

现在更强烈地阻止智能体重复打开浏览器标签页。当智能体尝试打开新标签页，且已有一个标签页打开到相同的主机时，除非智能体显式传递了标志，否则不会打开新标签页。

集成浏览器中的双指缩放（macOS）

集成浏览器 现在在 macOS 上支持双指缩放手势。使用触控板双指手势可以将网页内容放大至 3 倍。

与标准浏览器缩放（kb(workbench.action.browser.zoomIn) / kb(workbench.action.browser.zoomOut)）不同，双指缩放是纯粹的可视化放大，不会重新排版页面布局。

终端工具改进

此版本改进了智能体在后台运行终端命令的体验。

向后台终端发送输入

此前，后台终端是只读的，仅支持 get_terminal_output。这尤其限制了当前台终端超时并移至后台时，智能体无法再与其交互的情况。

借助新的 send_to_terminal 工具，智能体可以继续与后台终端交互。例如，如果 SSH 会话在等待密码提示时超时，智能体仍然可以发送所需的输入来完成连接。

后台终端通知（实验性）

设置: setting(chat.tools.terminal.backgroundNotifications)

此前，当终端命令在后台运行时，智能体必须手动调用 get_terminal_output 来检查其状态。无法知道命令何时完成或需要输入。

通过新的实验性 setting(chat.tools.terminal.backgroundNotifications) 设置，当后台终端命令完成或需要用户输入时，智能体会自动收到通知。这也适用于超时并移至后台的前台终端。然后，智能体可以采取适当的操作，例如审查输出或通过 send_to_terminal 工具提供输入。

已弃用的功能和设置

此版本中新增的弃用项

无

即将到来的弃用项

  • 编辑模式 自 VS Code 1.110 版本起已正式弃用。用户可以通过 VS Code 设置 setting(chat.editMode.hidden) 临时重新启用编辑模式。此设置将一直支持到 1.125 版本。从 1.125 版本开始，编辑模式将被完全移除，不再能通过设置启用。

    值得注意的修复

  • vscode#304257 - 集成 pwsh 的终端重启可能导致光标移动到错误的位置
  • vscode#304679 - Caps Lock 键在 VS Code 终端内的 Claude Code 中插入原始转义序列 "[57358u]"
