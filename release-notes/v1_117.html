Visual Studio Code 1.117

在 LinkedIn、X、Bluesky 上关注我们

_发布日期：2026 年 4 月 22 日_

欢迎来到 Visual Studio Code 1.117 版本。此版本为 Copilot Enterprise 和 Business 用户新增了功能，并进一步改善了 VS Code 中的代理体验。以下是本版本的亮点：

  • Copilot Business 和 Enterprise 的自带密钥：直接在 VS Code 聊天中连接你自己的 API 密钥，以使用偏好或专用的模型。
  • 聊天响应的增量渲染（实验性）：体验更流畅的聊天响应流式输出。
  • 终端改进：从任何终端配置文件启动 Copilot CLI。

祝编码愉快！

Copilot Business 和 Enterprise 的自带密钥

团队通常出于合规性、性能或成本原因需要特定的模型，但在不同工具之间切换使用会降低开发效率。自带语言模型密钥（BYOK）允许 Copilot Business 和 Enterprise 用户连接他们自己的 API 密钥，用于 OpenRouter、Ollama、Google、OpenAI 等提供商的模型，从而可以直接在 VS Code 聊天中使用这些模型。

默认情况下，BYOK 已启用，管理员可以通过 GitHub.com 上 Copilot 策略设置 中的 自带语言模型密钥 策略来禁用它。这使得管理员可以控制其组织中可用的模型提供商，同时让开发人员保持现有的工作流程。

策略启用后，组织成员可以从内置提供商添加模型或安装语言模型提供商扩展。

聊天体验

聊天响应的增量渲染（实验性）

通过增量渲染，聊天响应感觉更加流畅自然。增量渲染会在令牌到达时逐块流式输出内容，并可选用动画效果。与默认的基于计时器的渲染方式不同，这种实验性方法会在每个块准备就绪时立即渲染，减少了较长响应的感知等待时间。

使用以下设置配置增量响应渲染：

  • setting(chat.experimental.incrementalRendering.enabled)：启用或禁用增量响应渲染，在流式传输聊天响应时可选用块级动画。默认值：true。
  • setting(chat.experimental.incrementalRendering.animationStyle)：配置增量响应渲染的动画样式。选项：none、fade、rise、blur、scale、slide、reveal。默认值：fade。
  • setting(chat.experimental.incrementalRendering.buffering)：配置增量响应渲染中内容在渲染前的缓冲方式。较低的缓冲级别渲染更快，但可能显示不完整的句子或部分格式化的 Markdown。选项：off、word、paragraph。默认值：word。

    按最近活动排序代理会话

当你积累了许多代理会话时，找到正确的会话可能会很困难。代理会话 视图支持按创建时间或最后更新时间对会话进行排序，让你可以快速回到上次的工作进度。

后台终端命令的系统通知

当代理在后台运行长时间终端命令时，很容易忘记其进度。这些命令现在会在聊天响应中显示为系统通知，让你无需切换到终端即可监控其状态。

代理体验

Visual Studio Code Agents（Insiders）

注意：Visual Studio Code Agents 应用目前处于预览阶段，仅在安装 VS Code Insiders 时可用。

Visual Studio Code Agents 应用是与 VS Code Insiders 一起提供的配套应用，提供了一个专注于代理的原生环境，你可以在其中跨仓库运行并行会话、内联审查差异文件并迭代执行多步编码任务。该应用于 1.115 引入，并基于反馈持续演进。

本版本中的更新：

  • 创建子会话：在会话标题中选择 + 从当前会话创建子会话。这很方便用于在上下文中启动额外工作，例如并行研究或代码审查，而不会丢失在父会话中的工作进度。
  • 内联更改渲染：改进了更改的内联渲染方式，使代理编辑代码时更容易扫描和比较差异。
  • 更新体验：改进跨操作系统的更新流程，使保持最新版本更加顺畅。
  • 主题、聊天响应和用户体验优化：持续优化主题、会话列表和响应渲染以及整个应用的用户体验。

与之前版本相同，你可以通过以下方式打开该应用：

  • 从操作系统的开始菜单或应用程序文件夹中启动 Visual Studio Code Agents - Insiders。
  • 从 VS Code Insiders 命令面板中运行 Chat: 打开 Agents 应用。
  • 在 VS Code Insiders 欢迎页中选择 试用新的 Agents 应用。

    终端

    使用自定义终端配置文件启动 Copilot CLI

Copilot CLI 终端配置文件现在可以从终端面板启动，即使你的默认终端配置文件设置为非默认 shell，例如 macOS 或 Linux 上的 fish，或 Windows 上的 Git Bash。

此前，在此配置下从终端配置文件选择器中选择 GitHub Copilot CLI 会产生 未为 'copilot-cli' 提供终端配置文件选项 错误，导致终端无法启动。

代理 CLI 的终端标题

Copilot CLI、Claude Code 和 Gemini CLI 等代理 CLI 通常以 node 进程运行，这意味着终端标题会显示通用的 node 标签。这使得很难分辨每个终端中运行的是哪个代理。终端现在将这些代理 CLI 检测为独特的 shell 类型，并使用 CLI 发出的 OSC 标题序列作为终端标题，因此每个终端都能清楚地标识其承载的代理。

改进的检测功能覆盖 macOS、Linux 和 Windows 上的 Copilot CLI、Claude Code 和 Gemini CLI。Codex 目前在 macOS 上尚未被检测到，因为它目前不会发出 OSC 标题序列。此行为默认启用，可以通过 setting(terminal.integrated.tabs.allowAgentCliTitle) 设置进行切换。
